# skill name

requirement_pool

## purpose

把增长实验和漏斗问题转成可排期、可验收的需求池。

## input

- funnel output
- experiment output
- business priorities

## output

- requirement list
- priority tiers
- iteration phases
- acceptance criteria

## related agent

Requirement Agent

## evaluation criteria

- 是否能映射到具体实验或漏斗问题
- 是否有优先级和迭代阶段
- 是否定义验收标准
- 是否绑定影响指标

## example

输入：发布完成率低

输出：增加分类模板、简化表单、优化图片上传，优先级设为 P0，阶段为 MVP。
